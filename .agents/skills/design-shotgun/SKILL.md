# /design-shotgun — Design Exploration

"Show me options." Generate multiple visual directions for a component or page.

## Process

### 1. Understand the Target
What page or component needs design exploration? Read the relevant code.

### 2. Generate 4-6 Variants
Propose different visual directions. Describe each in detail:
- Layout approach (sidebar, top nav, card-based, etc.)
- Color treatment
- Typography choices
- Interaction patterns

### 3. Collect Feedback
Ask the user what they like and dislike about each variant. What would they combine from different variants?

### 4. Iterate
Refine the chosen direction based on feedback. Repeat until the user is satisfied.

### 5. Output
Write the final design direction to `.context/designs/shotgun-<target>.md`. Include: variants explored, feedback received, final direction, rationale.
